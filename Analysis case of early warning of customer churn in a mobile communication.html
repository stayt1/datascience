某移动通讯客户流失预警分析案例

姓名:杨涛

学号:1813580

案例背景：目前在我国移动通讯行业基本呈现三足鼎立的局势，市场份额由中国移动、中国联通和中国电信三家运营商瓜分。客户流失问题非常严重，从近5年的统计数字来看，三家运营商的移动客户数增长缓慢，中国联通在2015年甚至出现了负增长。传统的方法不仅成本高、准确度低，而且经常会打扰到正常客户，因此我希望开发一个系统的客户流失预警模型，来提前识别高风险流失客户。因此建立一套系统的客户流失预警体系已是燃眉之急。

  • 如图为参考链接1提供的图片：

展示了从2011年到2015年（近5年）中国移动、中国联通和中国电信的移动客户数增长趋势：三家运营商的移动客户数增长缓慢，中国联通在2015年时候出现了负增长。

  • 本案例使用的数据来自国内某运营商，数据已经进行了清理，数据集中的变量包括：
类型 变量名 详细说明 备注
因变量 是否流失 (churn) 1=流失；0=非流失 流失率1.27%2
自变量 在网时长（tenure） 连续变量，单位：天 客户从入网到截止数据提取日期时在网时间
当月花费（expense） 连续变量，单位：元 客户在提取月份时的花费总额
个体的度（degree） 连续变量, 单位： 人数 Di=∑j≠iaijDi=∑j≠iaij
联系强度（tightness） 通话总时间除以总人数，单位：分钟/人 Ti=Total−CommiDiTi=Total−CommiDi
个体信息熵（entropy） 连续变量 Ei=−∑aij=1pij∗log(pij)Ei=−∑aij=1pij∗log⁡(pij)
个体度的变化（chgdegree） 连续变量，单位：% （本月个体的度-上月个体的度）/上月个体的度3
花费的变化（chgexpense） 连续变量，单位：% （本月花费-上月花费）/上月花费

plot2 <- ggplot(sampledata,aes(x=churn,y=tightness,fill=churn))+geom_boxplot()+ylab("联系强度")+xlab("是否流失")
plot3 <- ggplot(sampledata,aes(x=churn,y=entropy,fill=churn))+geom_boxplot()+ylab("个体信息熵")+xlab("是否流失")
grid.arrange(plot1,plot2,plot3,nrow=1,ncol=3)
  • 由图中直观可以得到：
    • 可以看出个体的度和联系强度异常值越多，尾部越重，自由度越小。
    • 而且个体的信息熵异常值集中在较小值一侧，则分布呈左偏态；个体的度和联系强度异常值异常值集中在较大值一侧，则分布呈右偏态。
  • 具体分析：
    • 对于个体的度，平均来说流失客户的个体的度要小于非流失客户，说明要流失的客户可以明显看出通话行为的减少。
    • 对于联系强度，联系的强度在流失与非流失人群中的差异并不大，流失组的平均联系强度略低于低于非流失组。
    • 对于个体信息熵，流失组的个体信息熵平均要低于非流失组。个体的熵越小，说明用户联系的分布更加集中，那么流失的成本更低，更容易流失。

# Record boundary
list_mean <- apply(sampledata,2,mean)
list_sd <- apply(sampledata,2,sd)
list_max <- list_mean+3*(list_sd)
list_min <- list_mean-3*(list_sd)

Note

The 3σ3σ criterion is based on the equal-precision repeated measurement of the normal distribution, and the interference or noise of the singular data is difficult to meet the normal distribution. If the absolute value of the residual error of a measurement value in a set of measurement data νi>3σνi>3σ, then The measured value is a bad value and should be eliminated.

sub <- seq(0,1,0.025)
tol <- sum(preddata$churn)

  ss <- quantile(Yhat1, 1-s)

  res <- sum(preddata$churn[Yhat1 > ss])/tol
  return(res)
})
plot(sub,catch,col='red',type = 'l', xlab='覆盖率', ylab='捕获率')

任务五¶

  • 你对上述模型预测效果是否满意？在哪些方面做了进一步的探索？
    • 由以上的分析可以看出，进行逻辑回归的变量都是显著的，得到的模型也是显著的。
    • AUC值达到0.782，且Sensitivity和Specificity分别为72.4%和70.4%，模型预测的效果比较好。
    • 在覆盖率-捕获率曲线中可以看出，在设置20%的覆盖率的时候能够发现60%的可能流失客户，对于成本和方便的控制有极大的优势。
    • 一旦预测的流失概率超过了设定的阈值，预警体系就可以发出警告，告诉电信企业需要重点关注该客户。
  • 在哪些方面做了进一步的探索？
    • 在任务一时，异常值处理，使用3σ原则去除了对模型有影响的数据，使模型的预测效果更好。
    • 在任务二时，建立逻辑回归模型时候，使用AIC和BIC对变量进行进一步选择，但是发现选择之后所有的变量依然都是显著的，说明使用全部变量进行逻辑回归的想法是合理的。
    • 通过全模型的回归得到一些显著性的交互项，但是引入交互项之后使得AIC变大，而且模型的可解释性变差，故不作探索。
